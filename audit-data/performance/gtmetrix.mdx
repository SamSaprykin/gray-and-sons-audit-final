## Issues

### Important


- [ ] If we will take a look waterfall we will see page-data.json files which are prefetched pages data which gatsby generates based on all links. 
So if we have link to accessoires pages on the currect page - gatsby will add prerendered data. 


![waterfall-hp](/code/waterfall.png)

For example here https://fabulous-gecko-88393f.netlify.app/page-data/sterlingsilver/page-data.json - 712KB size. 

The main reason why this file is so large it's the way we render our images, e.g here how looks one image prerendere data:


![one-image-prefetch-data](/code/one-image-prefetch-data.png)

There are a total of 13 sizes for each image in 3 formats: webp, avif, and auto. I recommend reducing the number of sizes to 4 and removing webp (avif has better compression), as we have the default, which is the jpeg format.

https://caniuse.com/?search=avif


Greg, plesae take a look also this one: 

https://pagepro.co/blog/images-on-gatsby-website-vs-google-pagespeed-score/#6-gatsby-image-usage-with-additional-attributes

These recommendations look really useful:

fadeIn={false} – disables the fadein effect (no additional javascript action to top up the image)
loading=”eager” – set the browser’s native lazy attribute to eager
